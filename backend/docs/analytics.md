# Course Creator Analytics

This backend stores raw course analytics events and calculates creator metrics from a mix of raw events and existing canonical learning tables.

## Event Tracking

Student clients can write raw events with:

```http
POST /api/analytics/events
Authorization: Bearer <token>
Content-Type: application/json

{
  "courseId": 12,
  "lessonId": 44,
  "exerciseId": 103,
  "eventType": "question_answered",
  "correct": false,
  "durationSeconds": 18,
  "metadata": "{\"source\":\"lesson_player\"}"
}
```

Supported events:

- `course_enter`
- `course_exit`
- `lesson_start`
- `lesson_complete`
- `question_answered`

The backend also records:

- `lesson_complete` when `POST /api/courses/{courseId}/modules/{moduleId}/lessons/{lessonId}/complete` succeeds for the first time.
- `question_answered` when `POST /api/courses/{courseId}/modules/{moduleId}/lessons/{lessonId}/exercises/{exerciseId}/submit` succeeds.

For session and lesson time metrics, the frontend should send `durationSeconds` on `course_exit` and on explicit `lesson_complete` events.

## Database Schema

Flyway migration: `src/main/resources/db/migration/V8__analytics_events.sql`

```sql
CREATE TABLE analytics_events (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT NOT NULL,
    course_id BIGINT NOT NULL,
    lesson_id BIGINT,
    exercise_id BIGINT,
    event_type VARCHAR(50) NOT NULL,
    correct BOOLEAN,
    duration_seconds BIGINT,
    metadata TEXT,
    occurred_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP NOT NULL
);
```

Important indexes:

```sql
CREATE INDEX idx_analytics_events_course_time ON analytics_events (course_id, occurred_at);
CREATE INDEX idx_analytics_events_course_type_time ON analytics_events (course_id, event_type, occurred_at);
CREATE INDEX idx_analytics_events_user_course_time ON analytics_events (user_id, course_id, occurred_at);
```

Example records:

```sql
INSERT INTO analytics_events
    (user_id, course_id, lesson_id, exercise_id, event_type, correct, duration_seconds, metadata, occurred_at, created_at)
VALUES
    (5, 12, NULL, NULL, 'COURSE_ENTER', NULL, NULL, NULL, NOW(), NOW()),
    (5, 12, 44, NULL, 'LESSON_START', NULL, NULL, NULL, NOW(), NOW()),
    (5, 12, 44, 103, 'QUESTION_ANSWERED', FALSE, 18, '{"answer":"bonjour"}', NOW(), NOW()),
    (5, 12, 44, NULL, 'LESSON_COMPLETE', NULL, 420, NULL, NOW(), NOW()),
    (5, 12, NULL, NULL, 'COURSE_EXIT', NULL, 900, NULL, NOW(), NOW());
```

## Creator API

All analytics endpoints require authentication. The caller must be the course creator or an admin.

```http
GET /api/analytics/course/{id}/overview
GET /api/analytics/course/{id}/questions
GET /api/analytics/course/{id}/dropoff
```

## Metric Calculations

Daily and weekly active users use distinct users with any event in the course:

```sql
SELECT COUNT(DISTINCT user_id)
FROM analytics_events
WHERE course_id = :courseId
  AND occurred_at >= :since;
```

Average session duration uses `course_exit.duration_seconds`:

```sql
SELECT AVG(duration_seconds)
FROM analytics_events
WHERE course_id = :courseId
  AND event_type = 'COURSE_EXIT'
  AND duration_seconds IS NOT NULL;
```

Average total time spent per user sums tracked durations per user:

```sql
SELECT AVG(user_total)
FROM (
    SELECT user_id, SUM(duration_seconds) AS user_total
    FROM analytics_events
    WHERE course_id = :courseId
      AND duration_seconds IS NOT NULL
      AND event_type IN ('COURSE_EXIT', 'LESSON_COMPLETE')
    GROUP BY user_id
) totals;
```

Completion rate uses enrollments:

```sql
SELECT
    COUNT(*) FILTER (WHERE status = 'COMPLETED') * 100.0 / NULLIF(COUNT(*), 0) AS completion_rate
FROM enrollments
WHERE course_id = :courseId;
```

Average time to complete uses enrollment timestamps:

```sql
SELECT AVG(EXTRACT(EPOCH FROM (completed_at - enrolled_at))) AS avg_seconds
FROM enrollments
WHERE course_id = :courseId
  AND status = 'COMPLETED'
  AND completed_at IS NOT NULL;
```

Problematic questions are ranked by incorrect answers divided by all attempts:

```sql
SELECT e.id,
       e.lesson_id,
       e.question,
       COUNT(ea.id) AS total_answers,
       SUM(CASE WHEN ea.correct THEN 1 ELSE 0 END) AS correct_answers,
       SUM(CASE WHEN ea.correct THEN 0 ELSE 1 END) AS incorrect_answers,
       SUM(CASE WHEN ea.correct THEN 0 ELSE 1 END) * 100.0 / NULLIF(COUNT(ea.id), 0) AS error_rate
FROM exercises e
JOIN lessons l ON l.id = e.lesson_id
JOIN modules m ON m.id = l.module_id
LEFT JOIN exercise_attempts ea ON ea.exercise_id = e.id
WHERE m.course_id = :courseId
GROUP BY e.id, e.lesson_id, e.question
HAVING COUNT(ea.id) > 0
ORDER BY error_rate DESC, total_answers DESC;
```

Time spent per lesson uses explicit `lesson_complete.duration_seconds` events:

```sql
SELECT l.id,
       AVG(ae.duration_seconds) AS average_time_seconds,
       SUM(ae.duration_seconds) AS total_time_seconds
FROM lessons l
JOIN modules m ON m.id = l.module_id
LEFT JOIN analytics_events ae
    ON ae.lesson_id = l.id
   AND ae.event_type = 'LESSON_COMPLETE'
   AND ae.duration_seconds IS NOT NULL
WHERE m.course_id = :courseId
GROUP BY l.id
ORDER BY MIN(m.order_index), MIN(l.order_index);
```

Drop-off is calculated in backend logic by ordering lessons, finding each non-completed student's furthest completed lesson, and counting that lesson as their stopping point.
